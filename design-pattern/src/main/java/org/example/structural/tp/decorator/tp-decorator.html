Exercice : Enrichissement dynamique d’un service de notification

Contexte

Vous travaillez sur une application métier qui envoie des notifications aux utilisateurs. À l’origine, l’application envoie uniquement des notifications simples (texte brut).

Avec l’évolution du produit, de nouvelles exigences apparaissent :

  • journaliser certaines notifications,
  • chiffrer le contenu avant envoi,
  • mesurer le temps d’envoi,
  • combiner ces fonctionnalités selon les besoins.

Ces fonctionnalités doivent pouvoir être ajoutées ou retirées dynamiquement, sans modifier le service de notification existant et sans multiplier les sous-classes.

Problématique

Le service de notification doit rester :

  • simple,
  • stable,
  • centré uniquement sur son rôle principal : envoyer une notification.

Les fonctionnalités supplémentaires (log, chiffrement, métriques, etc.) ne doivent pas être intégrées directement dans le service principal.

Objectif

Mettre en place une solution basée sur le design pattern Decorator permettant d’enrichir dynamiquement le comportement d’un service de notification.

Description des éléments

Le système repose sur :

  • un service de notification chargé d’envoyer un message,
  • plusieurs fonctionnalités optionnelles pouvant enrichir ce service,
  • un client qui utilise le service de notification sans connaître les décorateurs appliqués.

Le service de notification de base :

  • envoie un message à un destinataire,
  • ne gère ni log, ni sécurité, ni mesure de performance.

Travail demandé

  1. Concevoir un service de notification de base.

  2. Mettre en place un ou plusieurs décorateurs permettant d’ajouter :

    • la journalisation des messages envoyés,
    • le chiffrement du contenu,
    • la mesure du temps d’exécution.

  3. Permettre au client de combiner librement ces fonctionnalités.

  4. Vérifier que le client n’a pas besoin d’être modifié lorsque de nouvelles fonctionnalités sont ajoutées.

Contraintes

  • Le service de notification de base ne doit pas être modifié.
  • Les fonctionnalités supplémentaires doivent être ajoutées dynamiquement.
  • Le pattern Decorator doit être respecté.
  • Le client doit manipuler un type unique représentant un service de notification.
